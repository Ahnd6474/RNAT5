# RNAT5

이전에는 분자의 의미를 표현하기 위한 공간만을 사용, 분자간의 상호작용을 표현하기 위한 공간이 필요함. 

세부사항: 
RNA 객체: AGUC의 파라미터화 된 좌표
파라미터: 여섯 개 backbone torsion(α, β, γ, δ, ε, ζ), base relative rotation인 χ, 그리고 sugar pucker 
메서드: 
get_coords: 
이전 residue의 frame/원자들 + 현재 residue의 torsion/pucker로 다음 residue 좌표를 결정하는 결정론적 forward kinematics
출력: RNA 분자의 좌표

더블 트랜스포머: RNA 객체 서열(RNA 서열을 미리 객체로 바꿔야 함, 파라미터는 기본값 사용)을 읽고 멀티 셀프 어텐션을 적용, 각 RNA 분자의 상태는 알지 못함
입력: RNA 서열(문자열)

병렬 어텐션: 각 분자의 문맥에 맞게 
상호작용 채널: 병렬 어텐션으로 산출한 각 토큰별 임베딩을 2d로 선형 사상하기
PE: 각 벡터에 Positional Embedding 더하기(i와 j간의 상호작용임을 알리기위해 2개를 다 쓰는 형식이 필요)
상호작용 벡터 병렬 어텐션: 
키/쿼리: 분자 벡터(1,j간 상호작용이면 i로 한번 j로 한번 업데이트)
발류: 상호작용벡터
분자 벡터 병렬 어텐션: 
키/쿼리: 상호작용벡터(i,j간 상호작용이면, 분자 벡터 i,j가 둘다 업데이트 )
발류: 분자벡터

상태 산출: i번째 분자 기준 
i와의 상호작용을 표현하는 모든 상호작용 벡터, i의 히든

좌표 계산: 출력 산출한 상태들을 좌표로 변환
RNA 객체의 get_coords를 반복 적용하되 'RNA 분자가 붙을 좌표'는 이전 RNA의 인산 좌표를 사용한다. 


훈련 방식: 내가 가진것은 PDB,리간드 서열이니 거기있는 걸로 손실 구성
훈련 대상: 더블 트랜스포머
